Staged Implementation Outline

Purpose
- Provide a clear, staged plan to implement the requested work. This document structures activities into Discover, Design, Execute, and Validate stages, with tasks, outputs, checkpoints, risks, timelines, and acceptance criteria.

Scope
- Applicable to a small-to-medium feature or project implementation produced by this agent. Adapt timelines and resource estimates as required.

Staged Plan

1) Discover (Goals: clarify requirements and constraints)
- Activities:
  - Gather and confirm requirements with stakeholders.
  - Inventory existing repository files and dependencies.
  - Identify assumptions, constraints, and success criteria.
  - Define minimal viable scope for an initial implementation (MVP).
- Deliverables:
  - Requirements summary (short document).
  - List of relevant files and environment notes.
- Checkpoints / Acceptance:
  - Stakeholder sign-off on requirements summary and MVP scope.
- Estimated duration: 0.5 - 2 days.
- Risks:
  - Unclear or changing requirements.

2) Design (Goals: produce a practical implementation plan)
- Activities:
  - Break the MVP into implementation tasks and subtasks.
  - Select architecture, modules, data/schema changes, and file paths.
  - Define interfaces, CLI/API signatures, and I/O files.
  - Prepare test plan: unit, integration, and validation tests.
- Deliverables:
  - Design document (architecture diagram, task breakdown).
  - Task backlog with priorities.
- Checkpoints / Acceptance:
  - Review of design against requirements; approval to proceed.
- Estimated duration: 0.5 - 3 days.
- Risks:
  - Overly ambitious scope; divergence from repo patterns.

3) Execute (Goals: implement the design and produce artifacts)
- Activities:
  - Implement tasks incrementally with small commits.
  - Produce code, scripts, configuration, and artifact files.
  - Create readable output files for review (notes, sample outputs).
  - Keep a changelog and record decisions in artifacts.
- Deliverables:
  - Source code and scripts in repo.
  - Artifact files (reports, sample outputs) under artifacts/.
  - Implementation summary explaining what was changed and why.
- Checkpoints / Acceptance:
  - Automated tests pass (where applicable).
  - Artifacts produced and reviewed by stakeholders.
- Estimated duration: variable (1 - 10+ days) depending on scope.
- Risks:
  - Integration issues, environment mismatches.

4) Validate (Goals: verify correctness and readiness)
- Activities:
  - Run automated tests and manual verification steps.
  - Compare outputs to expected results.
  - Produce a validation report with test results and unresolved issues.
  - If issues present, iterate: triage, fix, re-test.
- Deliverables:
  - Test results and validation report.
  - Final acceptance checklist.
- Checkpoints / Acceptance:
  - Stakeholder acceptance of validation report and sign-off.
- Estimated duration: 0.5 - 3 days.
- Risks:
  - Insufficient test coverage; late defect discovery.

Implementation Cadence and Milestones
- Milestone 1: Complete Discover & Design -> deliver requirements + design docs.
- Milestone 2: Deliver MVP implementation artifact and basic tests.
- Milestone 3: Complete Validation and stakeholder sign-off.

Deliverable File Layout (recommended)
- artifacts/staged_implementation_outline.md  # this file
- artifacts/requirements_summary.md
- artifacts/design_doc.md
- artifacts/implementation_summary.md
- artifacts/validation_report.md
- src/   # implementation code
- tests/ # automated tests and test cases

Quality and Validation
- Prefer small, reviewable commits and interim artifacts.
- Provide runnable examples and sample outputs.
- Include an acceptance checklist with pass/fail criteria.

Decisions and Constraints
- Aim for minimal external dependencies unless required.
- Keep artifacts under artifacts/ for discoverability.
- Use clear filenames and include README snippets for usage.
